# Internationalisation

This example keeps it simple and ships English only, but the surrounding ecosystem supports multilingual sites cleanly.

## Astra i18n directories

When `astra.config.json` declares additional locales, each locale gets its own subtree under `docs/`:

```json
{
  "i18n": {
    "defaultLocale": "en",
    "locales": [
      { "code": "en", "label": "English", "path": "" },
      { "code": "ja", "label": "日本語", "path": "ja" }
    ]
  }
}
```

With the config above:

- `docs/index.md` becomes `/`.
- `docs/ja/index.md` becomes `/ja/`.
- `docs/guide/intro.md` becomes `/guide/intro/`.
- `docs/ja/guide/intro.md` becomes `/ja/guide/intro/`.

Astra emits the appropriate `<html lang>` for each page based on which locale subtree the source file lives in. There is no per-page locale frontmatter to manage.

## PageFind multilingual indexes

PageFind autodetects language via the `<html lang>` attribute and builds a separate sub-index per detected language. Stemming, stopwords, and segmentation rules differ per language; for CJK languages PageFind uses a character-bigram strategy that performs well without per-language dictionaries.

When the user types a query, the JS API picks the matching sub-index based on the `<html lang>` of the page calling `pagefind.search()`. To search across all locales explicitly, pass `{ filters: { language: ["en", "ja"] } }` to `search()`.

The CLI does not need any extra flags — `pagefind --site dist` walks the tree, reads `<html lang>` from every HTML file, and bucketizes results.

## Things to watch for

- **Mixed-language pages.** If a single page mixes English and Japanese, PageFind classifies it by `<html lang>` only. Consider splitting such pages or wrapping the non-primary language in `<span lang="ja" data-pagefind-meta="…">` blocks.
- **Translated metadata.** The page `<title>` should be in the target language for each locale; the `description` frontmatter too. PageFind surfaces these as `result.meta.title` and `result.meta.description`.
- **Per-locale UI.** The `<search-box>` Web Component in this example uses an English placeholder. For a real multilingual deploy, read `document.documentElement.lang` and switch the placeholder string accordingly.

This example skips the locale tree entirely. The patterns above are what you would add when scaling up.
